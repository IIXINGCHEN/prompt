## 🎯 核心原则

### AxiomOS v15.0 五大核心原则
1. **领域驱动至上**: 所有决策以业务领域模型为核心
2. **规范驱动开发**: 严禁无规范的代码生成
3. **战略目标对齐**: 与项目宏观目标绝对一致
4. **完全可追溯性**: 每项产物可追溯到业务意图
5. **零信任安全**: 默认拒绝，严格认证授权

### 生产级交付物标准
- **A. 领域对齐**: 代码结构映射领域模型
- **B. 零信任安全**: 网络端点默认拒绝
- **C. 可靠性与弹性**: 幂等操作，优雅错误处理
- **D. 可观测性**: 结构化日志，标准指标
- **E. 可测试性**: 依赖注入，高覆盖率测试
- **F. 性能与效率**: 高效算法，避免性能陷阱
- **G. 可维护性**: SOLID原则，清晰注释

## 🚀 工作流程

### 6阶段工作流程
1. **Align (对齐)**: 模糊需求 → 精确规范
2. **Architect (架构)**: 共识文档 → 系统架构
3. **Atomize (原子化)**: 架构设计 → 原子任务
4. **Approve (批准)**: 设计确认与风险评估
5. **Act (执行)**: 生产级代码实现
6. **Assess (评估)**: 质量检查与验证

### 关键检查点
- 每个阶段都有明确的质量门控
- 必须获得用户明确批准才能进入下一阶段
- 所有交付物都要通过生产级标准验证

## 🔧 故障排除

### 常见问题
1. **MCP工具连接失败**
   - 检查Node.js版本兼容性
   - 验证网络连接和代理设置
   - 确认工具版本是否为最新

2. **协议执行异常**
   - 检查协议版本是否匹配
   - 验证配置文件格式正确性
   - 确认环境变量设置

3. **文档同步问题**
   - 检查Git仓库状态
   - 验证文件权限设置
   - 确认分支策略执行

### 调试工具
- 使用 `sequential-thinking` 进行问题分析
- 利用 `desktop-commander` 检查文件系统
- 通过 `context7` 查询相关文档
